# Component notes — PAM8403 audio amp (Heltec V4 only)

The **PAM8403** is a small class-D stereo amp board (Amazon B0DPMNYR2B) driving a
**4 Ω speaker** (B0F3DBRXS5). It is the sidetone output **only on the Heltec
V4** — the Cardputer ADV uses its onboard ES8311 codec via M5.Speaker instead
(see `cardputer-adv.md`), so none of this applies there.

## Wiring

- Powered from the Heltec **3V3 / 5V rail** (PAM8403 runs 2.5–5 V), common GND.
- Audio input from **GPIO4** (`PIN_SIDETONE`) through an **RC low-pass (~1 kΩ
  series + 100 nF)** into the amp's L_IN. The RC filter averages the PWM carrier
  back toward an analog waveform.
- Volume is set with the **amp board's hardware pot** — there is no software
  volume on the Heltec path (contrast the Cardputer, which has `setVolume`).

> GPIO4, not GPIO7. GPIO7 is the obvious audio pin but it is the **FEM power
> enable** on the V4 and must stay clear of the radio front-end. The sidetone was
> moved to GPIO4.

## Sidetone generation — lessons learned

The ESP32-S3 has **no true DAC**, so the tone is synthesized on the **LEDC** PWM
peripheral (`sidetone.cpp`). Tone frequency is **750 Hz** (`TONE_HZ`). There are
two code paths:

- **Square wave (`-DSIDETONE_SQUARE`) — the working default.** LEDC outputs a
  square wave; "volume" is the PWM duty cycle. This is what ships in the
  `heltec_v4` env. The RC filter is fitted.
- **PWM-sine (parked).** A hardware-timer ISR rewrites the LEDC duty at an audio
  sample rate (DDS) to trace a sine, with volume as the sine *amplitude* (so the
  timbre is constant at every level — the goal was a clean RSSI→loudness
  gradient). **This path did not drive the speaker** even with the RC filter
  fitted; the ISR duty-update sequence needs more bring-up. Parked behind the
  non-default branch until revisited.

### Why the volume curve must be exponential

The hunter maps received RSSI onto sidetone volume for an analog "tune for max
volume" feel. A **linear** map was inaudible: on a square wave nearly all the
loudness change sits in the flat top of the `sin(π·duty)` response, so equal RSSI
steps produced no perceived change. The working map is **dB-linear
(exponential)** — equal RSSI steps give equal perceived-loudness steps — from a
near-silent floor (duty 3) up to 50 % duty (loudest). The gradient is only
audible when RSSI actually varies, so run the fox at **LO** power.

## Mute

`sidetone_set_mute()` gates the LEDC output (writes duty 0 / parks at mid-rail)
while remembering the held key state, so unmute resumes a sounding element. It is
persisted (`config::muted`), applied at boot, and toggled by the BLE/serial
`mute` command. (The Cardputer mirrors the same API via `M5.Speaker.stop()`.)
